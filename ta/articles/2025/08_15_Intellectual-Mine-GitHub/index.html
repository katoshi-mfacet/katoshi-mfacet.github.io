அறிவுசார் சுரங்கமாக GitHub

15 ஆகஸ்ட், 2025

திறந்த மூல மென்பொருள் உருவாக்குநர்களிடையே கூட்டு மேம்பாட்டு தளமாகப் பயன்படுத்தப்படும் GitHub என்ற இணையச் சேவையைப் பற்றி நீங்கள் அறிந்திருக்கிறீர்களா?

சமீபத்திய ஆண்டுகளில், திறந்த மூல மென்பொருட்களுக்கு மட்டுமல்லாமல், நிறுவன மென்பொருள் மேம்பாடு மற்றும் மென்பொருள் அல்லாத நோக்கங்களுக்காகவும் இது ஒரு கூட்டுப் பணித் தளமாகப் பயன்படுத்தப்படுவது விரிவடைந்துள்ளது.

நான் எனது சொந்த நிரல்களையும், இந்த வலைப்பதிவிற்காக நான் எழுதும் கட்டுரைகளின் வரைவுகளையும் நிர்வகிக்க GitHub ஐப் பயன்படுத்துகிறேன்.

இந்தக் கட்டுரையில், GitHub இன் பயன்பாடு எதிர்காலத்தில் மென்பொருள் மேம்பாட்டிற்கு அப்பால் விரிவடைந்து, திறந்த அறிவுப் பகிர்வுக்கான இடமாக மாறும் சாத்தியக்கூறுகளை ஆராய்வேன்.

டீப்விக்கின் மூலம் விக்கி தளம் உருவாக்கம்

உருவாக்க AI ஐப் பயன்படுத்தும் பல மென்பொருள் மேம்பாட்டுக் கருவிகள் மனித நிரலாக்கப் பணிகளுக்கு உதவும் வகையில் வடிவமைக்கப்பட்டுள்ளன. மனிதர்கள் நிரல்களை எழுதுகிறார்கள், AI ஆதரவை வழங்குகிறது.

மறுபுறம், மனிதர்கள் வெறும் வழிமுறைகளை மட்டும் கொடுக்க, நிரல்களை உருவாக்கும் பணியை உருவாக்கும் AI எடுத்துக்கொள்ளும் ஒரு புதிய வகை மென்பொருள் மேம்பாட்டுக் கருவி உருவாகி வருகிறது.

டெவின் (Devin) அத்தகைய ஒரு கருவியாகும், இது ஒரு முன்னோடியாக மாறி கவனத்தைப் பெற்றது. டெவினை அறிமுகப்படுத்துவது மேம்பாட்டுக் குழுவில் மற்றொரு நிரலாளரைச் சேர்ப்பது போன்றது என்று சிலர் கூறினர். இதை திறம்படப் பயன்படுத்த மனிதப் பொறியாளர்கள் விரிவான ஆதரவை வழங்க வேண்டும் என்று இன்னும் கூறப்பட்டாலும், அத்தகைய தரவுகள் நிச்சயமாகச் சேகரிக்கப்பட்டு மேம்பாட்டிற்குப் பயன்படுத்தப்படும்.

ஒரு மனிதனையும், டெவின் போன்ற AI நிரலாளர்களையும் கொண்ட மென்பொருள் மேம்பாட்டுக் குழுக்கள் சாதாரணமாக மாறும் காலம் வெகு தொலைவில் இல்லை.

டெவின் இன் உருவாக்குநரான காக்னிஷன் (Cognition), டீப்விக்கி (DeepWiki) என்ற சேவையையும் வெளியிட்டுள்ளது.

டீப்விக்கி என்பது GitHub இல் உள்ள ஒவ்வொரு மென்பொருள் மேம்பாட்டுத் திட்டத்திற்கும் ஒரு விக்கி தளத்தை தானாக உருவாக்கும் ஒரு சேவையாகும். அதாவது, டெவின் போன்ற ஒரு AI, அந்தத் திட்டத்தின் அனைத்து நிரல்களையும் தொடர்புடைய ஆவணங்களையும் படித்து பகுப்பாய்வு செய்து, அனைத்து கையேடுகளையும் வடிவமைப்பு ஆவணங்களையும் உருவாக்குகிறது.

காக்னிஷன் நிறுவனம், GitHub இல் யார் வேண்டுமானாலும் இலவசமாக அணுகக்கூடிய 50,000 க்கும் மேற்பட்ட முக்கிய பொது மென்பொருள் மேம்பாட்டுத் திட்டங்களுக்காக டீப்விக்கைப் பயன்படுத்தி விக்கி தளங்களை உருவாக்கியதாகக் கூறப்படுகிறது.

இவை பொதுத் திட்டங்கள் என்பதால், இதைச் செய்வதில் எந்தப் பிரச்சனையும் இல்லை. விக்கி தளங்களை தானாக உருவாக்க முடியும் என்றாலும், ஏராளமான உருவாக்க AI கள் நீண்ட காலத்திற்கு முழுத் திறனில் இயங்க வேண்டியிருந்திருக்கும், மேலும் அதன் செலவும் கணிசமாக இருந்திருக்கும்.

இந்தச் செலவுகளை ஏற்றுக்கொண்டதன் மூலம், காக்னிஷன் நிறுவனம் ஏராளமான பொதுத் திட்டங்களுக்கு ஒரு பெரிய பலனை அளித்துள்ளது, அவர்களுக்கு விளக்கங்களையும் வடிவமைப்பு ஆவணங்களையும் இலவசமாகப் பெற அனுமதித்துள்ளது.

இந்த விக்கி தளங்கள் ஒவ்வொரு பொதுத் திட்டத்திற்கும் பயனுள்ளதாக இருக்கின்றன என்றும், தரம் மற்றும் உற்பத்தித்திறனை மேம்படுத்துவதில் குறிப்பிடத்தக்க விளைவைக் கொண்டுள்ளன என்றும் புள்ளிவிவரத் தரவுகள் காட்டினால், மென்பொருள் மேம்பாட்டு நிறுவனங்கள் தங்கள் சொந்த திட்டங்களுக்கு டீப்விக்கைத் தத்தெடுக்கும்.

இது நிகழக்கூடும் என்று நம்பி, ஏராளமான பொதுத் திட்டங்களுக்கான விக்கி தளங்களை உருவாக்குவதில் காக்னிஷன் முதலீடு செய்திருக்க வேண்டும். இது டீப்விக்கி மீதான காக்னிஷன் இன் நம்பிக்கையை வெளிப்படுத்துகிறது. டீப்விக்கி ஏற்றுக்கொள்ளப்படும்போது, டெவின் தானாகவே அதைப் பின்தொடரும், AI நிரலாளர்களை பரவலாக ஏற்றுக்கொள்வதற்கான வாய்ப்பை கணிசமாக அதிகரிக்கும்.

ஆவணப் பகிர்வு தளமாக GitHub

GitHub ஆனது, திறந்த மூல மென்பொருள் மேம்பாட்டிற்கான நிரல்களைப் பகிர்தல், கூட்டாகத் திருத்துதல் மற்றும் சேமிப்பதற்கான ஒரு பிரபலமான மற்றும் நடைமுறை நிலையான இணைய சேவையாக மாறியுள்ளது.

சமீபத்திய ஆண்டுகளில், நிறுவனங்களுக்கான அதன் மேலாண்மை மற்றும் பாதுகாப்பு அம்சங்கள் மேம்படுத்தப்பட்டுள்ளன, இது மென்பொருளை மேம்படுத்தும் மேம்பட்ட நிறுவனங்களில் ஒரு பொதுவான கருவியாக மாற்றியுள்ளது.

இந்தக் காரணத்திற்காக, GitHub ஆனது நிரல்களைச் சேமிப்பதற்கும் பகிர்வதற்கும் ஒரு இணைய சேவை என்ற பிம்பத்தை வலிமையாகத் தூண்டுகிறது. இருப்பினும், உண்மையில், இது நிரல்களுடன் முற்றிலும் தொடர்பில்லாத பல்வேறு ஆவணங்கள் மற்றும் பொருட்களைப் பகிர, கூட்டாகத் திருத்த மற்றும் சேமிக்கப் பயன்படுத்தப்படலாம்.

எனவே, பரவலாக கூட்டாகத் திருத்த விரும்பும் ஆவணங்களை நிர்வகிக்க GitHub ஐப் பயன்படுத்துபவர்கள் சிலர் அல்ல. இவை மென்பொருள் தொடர்பான ஆவணங்களாகவோ அல்லது முற்றிலும் தொடர்பில்லாத ஆவணங்களாகவோ இருக்கலாம்.

மேலும், வலைப்பதிவுகள் மற்றும் வலைத்தளங்கள் ஒரு வகையான நிரலைக் கொண்டிருக்கும் அல்லது நிரல்களால் கட்டமைக்கப்பட்டு வெளியிடப்படும் ஆவணங்களாகும்.

இதன் காரணமாக, தனிநபர்கள் மற்றும் நிறுவனங்கள் வலைப்பதிவுகள் மற்றும் வலைத்தளங்களின் உள்ளடக்கத்தை, அவற்றை எளிதாகப் பார்க்க உதவும் நிரல்கள் மற்றும் தானியங்கு தள உருவாக்கத்திற்கான நிரல்களுடன் சேர்த்து, GitHub இல் ஒரே திட்டமாக சேமிப்பது வழக்கத்திற்கு மாறானதல்ல.

அத்தகைய வலைப்பதிவுகள் மற்றும் வலைத்தளங்களின் உள்ளடக்கத்தை கூட்டாகத் திருத்துவதற்காக GitHub இல் பொதுத் திட்டங்களாக மாற்றுவதும் சாத்தியமாகும்.

மேலும், சமீபத்தில், உருவாக்க AI ஆனது மென்பொருள் மேம்பாட்டிற்கு மட்டுமல்லாமல், மென்பொருளில் அடிக்கடி ஒருங்கிணைக்கப்படுகிறது.

இந்தச் சந்தர்ப்பத்தில், உருவாக்க AI க்கு விரிவான வழிமுறைகளை வழங்கும் தூண்டுதல்கள் எனப்படும் அறிவுறுத்தல் வாக்கியங்கள் நிரல்களுக்குள் உட்பொதிக்கப்படுகின்றன.

இந்தத் தூண்டுதல்கள் ஒரு வகை ஆவணமாகவும் கருதப்படலாம்.

அறிவுசார் தொழிற்சாலை

நான் ஒரு மென்பொருள் மேம்பாட்டுப் பொறியாளர் என்றாலும், எனது வலைப்பதிவிற்கான கட்டுரைகளையும் எழுதுகிறேன்.

பலர் அவற்றைப் படிக்க வேண்டும் என்று நான் விரும்பினாலும், வாசகர்களின் எண்ணிக்கையை அதிகரிப்பது மிகவும் கடினம்.

நிச்சயமாக, கவனத்தை ஈர்க்கும் கட்டுரைகளை உருவாக்குவது அல்லது செல்வாக்கு மிக்க நபர்களை ஆலோசனைக்காக தீவிரமாகத் தொடர்புகொள்வது போன்ற பிற முயற்சிகளையும் புத்திசாலித்தனத்தையும் ஒருவர் பரிசீலிக்கலாம்.

இருப்பினும், எனது ஆளுமையையும், இதில் சம்பந்தப்பட்ட முயற்சி மற்றும் மன அழுத்தத்தையும் கருத்தில் கொண்டு, தீவிரமான விளம்பரங்களில் ஈடுபட நான் தயங்குகிறேன். மேலும், அத்தகைய நடவடிக்கைகளில் நேரம் செலவிடுவது எனது வேலையின் முக்கிய அம்சத்திலிருந்து திசைதிருப்பும், அது நிரலாக்கம், யோசனைகளைக் கருத்தில் கொள்வது மற்றும் அவற்றை ஆவணப்படுத்துவது ஆகும்.

எனவே, எனது வலைப்பதிவு இடுகைகளின் வரம்பை பல்வேறு வகையான உள்ளடக்கங்களாக மேம்படுத்துவதன் மூலம் விரிவாக்கும், மல்டிமீடியா அல்லது ஓம்னிசேனல் (omnichannel) எனப்படும் ஒரு உத்தியை முயற்சிக்க சமீபத்தில் நான் முடிவு செய்தேன்.

குறிப்பாக, ஜப்பானியக் கட்டுரைகளை ஆங்கிலத்தில் மொழிபெயர்த்து ஆங்கில வலைப்பதிவு தளங்களில் இடுகையிடுவது, மற்றும் கட்டுரைகளை விளக்கும் விளக்கக்காட்சி வீடியோக்களை உருவாக்கி YouTube இல் வெளியிடுவது ஆகியவை இதில் அடங்கும்.

மேலும், பொது வலைப்பதிவு சேவைகளில் வெளியிடுவதுடன், எனது கடந்தகால வலைப்பதிவு இடுகைகளை பட்டியலிட்டு வகைப்படுத்தும் மற்றும் தொடர்புடைய கட்டுரைகளை இணைக்கும் எனது சொந்த வலைப்பதிவு தளத்தை உருவாக்குவதையும் நான் பரிசீலித்து வருகிறேன்.

ஒரு புதிய கட்டுரை எழுதப்படும் ஒவ்வொரு முறையும் இவற்றை உருவாக்க நான் நேரம் செலவழித்தால், அது எதிர்மறையான விளைவுகளை ஏற்படுத்தும். எனவே, ஆரம்ப ஜப்பானியக் கட்டுரையை எழுதுவதைத் தவிர மற்ற அனைத்துப் பணிகளும் உருவாக்கும் AI ஐப் பயன்படுத்தி தானியங்குபடுத்தப்படுகின்றன. இதை நான் ஒரு அறிவுசார் தொழிற்சாலை என்று அழைக்கிறேன்.

இந்த வழிமுறையைச் செயல்படுத்த நிரல்களை உருவாக்க வேண்டும்.

தற்போது, மொழிபெயர்ப்பு, விளக்கக்காட்சி வீடியோ உருவாக்கம் மற்றும் YouTube இல் பதிவேற்றுதல் ஆகியவற்றை முழுமையாக தானியங்குபடுத்தக்கூடிய நிரல்களை நான் ஏற்கனவே உருவாக்கியுள்ளேன்.

இப்போது இருக்கும் வலைப்பதிவு இடுகைகளை வகைப்படுத்துவதற்கும் இணைப்பதற்கும் அடிப்படை நிரல்களை உருவாக்கும் பணியில் நான் இருக்கிறேன்.

அது முடிந்ததும், எனது சொந்த வலைப்பதிவு தளத்தை உருவாக்கவும், அதை ஒரு வலை சேவையகத்தில் தானாகப் பிரதிபலிக்கவும் ஒரு நிரலை உருவாக்கினால், எனது அறிவுசார் தொழிற்சாலையின் ஆரம்பக் கருத்து நிறைவடையும்.

பரந்த பொருளில் அறிவுசார் தொழிற்சாலை

இந்த அறிவுசார் தொழிற்சாலைக்கான மூலப்பொருளாகச் செயல்படும் எனது வலைப்பதிவு இடுகைகளின் வரைவுகள், ஒரு GitHub திட்டமாகவும் நிர்வகிக்கப்படுகின்றன. இப்போதைக்கு, அவை தனிப்பட்டவை மற்றும் பொதுவில் கிடைக்கவில்லை, ஆனால் எதிர்காலத்தில் அறிவுசார் தொழிற்சாலை நிரல்களுடன் சேர்த்து அவற்றை பொதுத் திட்டங்களாக மாற்றுவதைப் பரிசீலித்து வருகிறேன்.

மேலும், நான் தற்போது உருவாக்கி வரும் வலைப்பதிவு இடுகைகளின் வகைப்படுத்தல், கட்டுரைகளை இணைத்தல், மற்றும் வீடியோவாக மாற்றப்பட்ட வலைப்பதிவு இடுகைகளின் விளக்கம் ஆகியவை DeepWiki ஐப் போன்ற அடிப்படை கருத்தைக் கொண்டுள்ளன.

உருவாக்கும் AI ஐப் பயன்படுத்தி, அசல் படைப்புப் பணிகளிலிருந்து மூலப்பொருட்களாகப் பல்வேறு உள்ளடக்கங்கள் தயாரிக்கப்படுகின்றன. கூடுதலாக, இது தகவல்களையும் அறிவையும் அவற்றுக்குள் இணைக்க முடியும், திறம்பட ஒரு அறிவுத் தளத்தை உருவாக்குகிறது.

ஒரே வித்தியாசம் என்னவென்றால், மூலப்பொருள் ஒரு நிரலா அல்லது வலைப்பதிவு இடுகையா என்பதுதான். மேலும் DeepWiki மற்றும் உருவாக்கும் AI ஆல் இயங்கும் எனது அறிவுசார் தொழிற்சாலைக்கு, அந்த வேறுபாடு கிட்டத்தட்ட அர்த்தமற்றது.

வேறு வார்த்தைகளில் கூறுவதானால், "அறிவுசார் தொழிற்சாலை" என்ற சொல் எனது நிரலுடன் மட்டும் மட்டுப்படுத்தப்படாமல், ஒரு பொதுவான, பரந்த பொருளில் விளக்கப்படும்போது, DeepWiki யும் ஒரு வகையான அறிவுசார் தொழிற்சாலையாகும்.

மேலும், அறிவுசார் தொழிற்சாலைகள் தயாரிப்பது மற்ற மொழிகளில் மொழிபெயர்க்கப்பட்ட கட்டுரைகள், விளக்கக்காட்சி வீடியோக்கள், சுயமாக உருவாக்கப்பட்ட வலைப்பதிவு தளங்கள் அல்லது விக்கி தளங்கள் மட்டுமல்ல.

குறுகிய வீடியோக்கள், ட்வீட்கள், காமிக்ஸ், அனிமேஷன், பாட்காஸ்ட்கள் மற்றும் மின் புத்தகங்கள் போன்ற கற்பனை செய்யக்கூடிய ஒவ்வொரு ஊடகம் மற்றும் வடிவமாகவும் உள்ளடக்கத்தை மாற்ற அவை வாய்ப்புள்ளது.

மேலும், இந்த ஊடகங்கள் மற்றும் வடிவங்களுக்குள் உள்ள உள்ளடக்கம் பெறுநருக்கு ஏற்றவாறு பல்வகைப்படுத்தப்படலாம், அதாவது பரந்த பல மொழி ஆதரவு, வல்லுநர்கள் அல்லது ஆரம்பநிலையாளர்களுக்கான பதிப்புகள், மற்றும் பெரியவர்கள் அல்லது குழந்தைகளுக்கான பதிப்புகள்.

மேலும், தனிப்பயனாக்கப்பட்ட உள்ளடக்கத்தை தேவைக்கேற்ப உருவாக்குவதும் சாத்தியமாகும்.

அறிவுசார் சுரங்கமாக கிட்ஹப் (GitHub)

ஒரு அறிவுசார் தொழிற்சாலைக்கான மூலப்பொருட்கள் அடிப்படையில் எங்கு வேண்டுமானாலும் அமைந்திருக்கலாம்.

இருப்பினும், கிட்ஹப் (GitHub) ஆனது திறந்த மூல திட்ட நிரல்களைப் பகிர்தல், கூட்டாகத் திருத்துதல் மற்றும் சேமிப்பதற்கான நடைமுறைத் தரமாக மாறியுள்ளது என்பதையும், நானும் மட்டுமல்லாமல் பலரும் கிட்ஹப்பை ஒரு ஆவணச் சேமிப்பு இடமாகப் பயன்படுத்துகிறார்கள் என்பதையும் கருத்தில் கொள்ளும்போது, கிட்ஹப் அறிவுசார் தொழிற்சாலைகளுக்கான மூலப்பொருட்களின் முதன்மை ஆதாரமாக மாறும் ஆற்றலைக் கொண்டுள்ளது என்பது தெளிவாகிறது.

வேறு வார்த்தைகளில் கூறுவதானால், கிட்ஹப் மனிதகுலத்திற்கான பகிரப்பட்ட அறிவுசார் சுரங்கமாக மாறி, அறிவுசார் தொழிற்சாலைகளுக்கு மூலப்பொருட்களை வழங்கும்.

இங்கு "மனிதகுலத்தால் பகிரப்பட்டது" என்ற சொல், திறந்த மூல திட்டங்கள் மனிதகுலத்திற்கான பகிரப்பட்ட மென்பொருள் சொத்து என்ற கருத்தை எதிரொலிக்கிறது.

கிட்ஹப்பை ஆதரித்த திறந்த மூல தத்துவம், திறந்த ஆவணங்களின் கருத்தாக்கத்திற்கும் பொருந்தும்.

மேலும், நிரல்களைப் போலவே, ஒவ்வொரு ஆவணத்திற்கும் பதிப்புரிமைத் தகவல் மற்றும் உரிமங்களை நிர்வகிக்கும் கலாச்சாரம் உருவாகலாம். மூல ஆவணங்களிலிருந்து தானாக உருவாக்கப்படும் உள்ளடக்கத்திற்கு அதே உரிமத்தை எளிதாக ஒதுக்கலாம் அல்லது உரிமத்தால் குறிப்பிடப்பட்டுள்ள விதிகளுக்கு இணங்கலாம்.
