GitHub ในฐานะเหมืองทางปัญญา

15 สิงหาคม 2568

คุณคุ้นเคยกับ GitHub ซึ่งเป็นบริการเว็บที่ใช้เป็นแพลตฟอร์มการพัฒนาร่วมกันในหมู่นักพัฒนาซอฟต์แวร์โอเพนซอร์สหรือไม่?

ในช่วงไม่กี่ปีที่ผ่านมา การใช้งานในฐานะแพลตฟอร์มสำหรับการทำงานร่วมกันได้ขยายตัว ไม่เพียงแต่สำหรับซอฟต์แวร์โอเพนซอร์สเท่านั้น แต่ยังรวมถึงการพัฒนาซอฟต์แวร์ขององค์กร และแม้กระทั่งวัตถุประสงค์ที่ไม่เกี่ยวข้องกับซอฟต์แวร์อีกด้วย

ฉันยังใช้ GitHub เพื่อจัดการโปรแกรมของตัวเองและฉบับร่างบทความที่ฉันเขียนสำหรับบล็อกนี้

ในบทความนี้ ฉันจะสำรวจความเป็นไปได้ที่การใช้งาน GitHub จะขยายออกไปนอกเหนือจากการพัฒนาซอฟต์แวร์มากขึ้นในอนาคต กลายเป็นสถานที่สำหรับการแบ่งปันความรู้แบบเปิด

การสร้างเว็บไซต์ Wiki ด้วย DeepWiki

เครื่องมือพัฒนาซอฟต์แวร์จำนวนมากที่ใช้ AI สร้างสรรค์ถูกออกแบบมาเพื่อช่วยเหลืองานการเขียนโปรแกรมของมนุษย์ โดยที่มนุษย์เป็นผู้เขียนโปรแกรม และ AI ให้การสนับสนุน

ในทางกลับกัน เครื่องมือพัฒนาซอฟต์แวร์ประเภทใหม่กำลังเกิดขึ้น โดยที่มนุษย์เพียงให้คำแนะนำ และ AI สร้างสรรค์จะเข้ามารับหน้าที่ในการสร้างโปรแกรม

Devin เป็นหนึ่งในเครื่องมือดังกล่าวที่เป็นผู้บุกเบิกและได้รับความสนใจ บางคนถึงกับกล่าวว่าการแนะนำ Devin เข้ามาก็เหมือนกับการเพิ่มโปรแกรมเมอร์อีกหนึ่งคนเข้าสู่ทีมพัฒนา แม้จะยังคงมีการกล่าวถึงว่าวิศวกรมนุษย์จำเป็นต้องให้การสนับสนุนอย่างละเอียดเพื่อให้สามารถใช้งานได้อย่างมีประสิทธิภาพ แต่ข้อมูลดังกล่าวจะถูกรวบรวมและนำไปใช้ในการปรับปรุงอย่างแน่นอน

ยุคที่ทีมพัฒนาซอฟต์แวร์ประกอบด้วยมนุษย์หนึ่งคนและโปรแกรมเมอร์ AI อย่าง Devin เป็นสมาชิกทีมจะกลายเป็นเรื่องธรรมดาในไม่ช้า

Cognition ผู้พัฒนา Devin ได้เปิดตัวบริการชื่อ DeepWiki ด้วย

DeepWiki เป็นบริการที่สร้างเว็บไซต์ Wiki โดยอัตโนมัติสำหรับโครงการพัฒนาซอฟต์แวร์แต่ละโครงการบน GitHub ซึ่งหมายความว่า AI ที่คล้ายกับ Devin จะอ่านและวิเคราะห์โปรแกรมทั้งหมดและเอกสารที่เกี่ยวข้องของโครงการนั้น ๆ และสร้างคู่มือและเอกสารการออกแบบทั้งหมด

Cognition รายงานว่าได้สร้างเว็บไซต์ Wiki สำหรับโครงการพัฒนาซอฟต์แวร์สาธารณะหลัก ๆ กว่า 50,000 โครงการบน GitHub ที่ทุกคนสามารถเข้าถึงได้อย่างอิสระ โดยใช้ DeepWiki

เนื่องจากเป็นโครงการสาธารณะ จึงไม่มีปัญหาใด ๆ ในการทำเช่นนั้น แม้ว่าเว็บไซต์ Wiki สามารถสร้างได้โดยอัตโนมัติ แต่ก็ต้องใช้ AI สร้างสรรค์จำนวนมากในการทำงานเต็มกำลังเป็นเวลานาน และค่าใช้จ่ายก็น่าจะสูงมาก

ด้วยการแบกรับค่าใช้จ่ายเหล่านี้ Cognition ได้มอบประโยชน์อย่างใหญ่หลวงให้กับโครงการสาธารณะจำนวนมาก ทำให้พวกเขาสามารถได้รับคำอธิบายและเอกสารการออกแบบได้ฟรี

หากข้อมูลทางสถิติแสดงให้เห็นว่าเว็บไซต์ Wiki เหล่านี้มีประโยชน์ต่อโครงการสาธารณะแต่ละโครงการและมีผลกระทบอย่างมีนัยสำคัญในการปรับปรุงคุณภาพและประสิทธิภาพ บริษัทพัฒนาซอฟต์แวร์ก็จะนำ DeepWiki ไปใช้กับโครงการของตนเอง

Cognition ต้องลงทุนในการสร้างเว็บไซต์ Wiki สำหรับโครงการสาธารณะจำนวนมาก โดยเชื่อว่าสิ่งนี้จะเกิดขึ้นได้ ซึ่งแสดงให้เห็นถึงความมั่นใจของ Cognition ใน DeepWiki และเมื่อ DeepWiki ถูกนำมาใช้ Devin ก็จะตามมาโดยอัตโนมัติ ซึ่งจะเพิ่มความเป็นไปได้ในการนำโปรแกรมเมอร์ AI มาใช้อย่างแพร่หลายอย่างมีนัยสำคัญ

GitHub ในฐานะแพลตฟอร์มการแบ่งปันเอกสาร

GitHub ได้กลายเป็นบริการเว็บยอดนิยมและเป็นมาตรฐานโดยพฤตินัยสำหรับการแบ่งปัน การแก้ไขร่วมกัน และการจัดเก็บโปรแกรมสำหรับการพัฒนาซอฟต์แวร์โอเพนซอร์ส

ในช่วงไม่กี่ปีที่ผ่านมา คุณสมบัติการจัดการและความปลอดภัยสำหรับองค์กรได้รับการปรับปรุง ทำให้เป็นเครื่องมือทั่วไปในบริษัทชั้นนำที่พัฒนาซอฟต์แวร์

ด้วยเหตุนี้ GitHub จึงสร้างภาพลักษณ์ของบริการเว็บสำหรับการจัดเก็บและแบ่งปันโปรแกรมอย่างมาก อย่างไรก็ตาม ในความเป็นจริงแล้ว สามารถใช้เพื่อแบ่งปัน แก้ไขร่วมกัน และจัดเก็บเอกสารและวัสดุต่างๆ ที่ไม่เกี่ยวข้องกับโปรแกรมได้เลย

ดังนั้น จึงมีผู้คนจำนวนไม่น้อยที่ใช้ GitHub เพื่อจัดการเอกสารที่พวกเขาต้องการแก้ไขร่วมกันอย่างกว้างขวาง ซึ่งอาจเป็นเอกสารที่เกี่ยวข้องกับซอฟต์แวร์หรือไม่เกี่ยวข้องเลยก็ได้

ยิ่งไปกว่านั้น บล็อกและเว็บไซต์ก็เป็นเอกสารที่อาจมีโปรแกรมบางประเภท หรือถูกจัดโครงสร้างและเผยแพร่โดยโปรแกรม

ด้วยเหตุนี้ จึงไม่ใช่เรื่องแปลกสำหรับบุคคลและบริษัทที่จะจัดเก็บเนื้อหาของบล็อกและเว็บไซต์ พร้อมกับโปรแกรมที่ทำให้ดูง่าย และโปรแกรมสำหรับการสร้างเว็บไซต์อัตโนมัติ รวมกันเป็นโครงการเดียวบน GitHub

นอกจากนี้ยังสามารถทำให้บล็อกและเว็บไซต์ดังกล่าวเป็นโครงการสาธารณะบน GitHub เพื่อการแก้ไขเนื้อหาร่วมกันได้อีกด้วย

ยิ่งไปกว่านั้น เมื่อเร็วๆ นี้ AI สร้างสรรค์ไม่ได้ถูกนำมาใช้สำหรับการพัฒนาซอฟต์แวร์เท่านั้น แต่ยังมักถูกรวมเข้ากับซอฟต์แวร์ด้วย

ในกรณีนี้ ประโยคคำสั่งที่เรียกว่าพรอมต์ ซึ่งให้คำแนะนำโดยละเอียดแก่ AI สร้างสรรค์ จะถูกฝังอยู่ในโปรแกรม

พรอมต์เหล่านี้ก็ถือเป็นเอกสารประเภทหนึ่งได้เช่นกัน

โรงงานทางปัญญา

แม้ว่าฉันจะเป็นวิศวกรพัฒนาซอฟต์แวร์ แต่ฉันก็เขียนบทความสำหรับบล็อกของฉันด้วย

แม้ว่าฉันต้องการให้หลายคนอ่านบทความเหล่านี้ แต่การเพิ่มจำนวนผู้อ่านนั้นค่อนข้างยาก

แน่นอนว่าเราสามารถพิจารณาสร้างบทความเพื่อดึงดูดความสนใจ หรือติดต่อบุคคลผู้ทรงอิทธิพลเพื่อขอคำแนะนำ รวมถึงความพยายามและความคิดสร้างสรรค์อื่นๆ อีกมากมาย

อย่างไรก็ตาม เมื่อพิจารณาถึงบุคลิกของฉัน รวมถึงความพยายามและความเครียดที่เกี่ยวข้อง ฉันไม่เต็มใจที่จะเข้าร่วมในการโปรโมตเชิงรุก ยิ่งไปกว่านั้น การใช้เวลากับกิจกรรมดังกล่าวจะทำให้เสียเวลาจากแก่นแท้ของงานของฉัน ซึ่งเกี่ยวข้องกับการเขียนโปรแกรม การใคร่ครวญแนวคิด และการจัดทำเอกสาร

ดังนั้น เมื่อเร็วๆ นี้ฉันจึงตัดสินใจลองใช้กลยุทธ์ที่เรียกว่ามัลติมีเดีย หรือ Omnichannel ซึ่งเกี่ยวข้องกับการขยายขอบเขตการเข้าถึงบทความบล็อกของฉันโดยการพัฒนาบทความเหล่านั้นให้เป็นเนื้อหารูปแบบต่างๆ

โดยเฉพาะอย่างยิ่ง ซึ่งรวมถึงการแปลบทความภาษาญี่ปุ่นเป็นภาษาอังกฤษและโพสต์ลงในเว็บไซต์บล็อกภาษาอังกฤษ และการสร้างวิดีโอพรีเซนเทชันเพื่ออธิบายบทความและเผยแพร่บน YouTube

นอกจากนี้ นอกจากการเผยแพร่บนบริการบล็อกทั่วไปแล้ว ฉันยังพิจารณาสร้างเว็บไซต์บล็อกของตัวเองที่แสดงรายการและจัดหมวดหมู่บทความบล็อกที่ผ่านมาของฉัน และเชื่อมโยงบทความที่เกี่ยวข้องอีกด้วย

หากฉันต้องใช้เวลาในการสร้างสิ่งเหล่านี้ทุกครั้งที่มีบทความใหม่ การกระทำดังกล่าวจะขัดต่อวัตถุประสงค์ ดังนั้น งานทั้งหมดนอกเหนือจากการเขียนบทความภาษาญี่ปุ่นฉบับเริ่มต้นจะถูกทำให้เป็นอัตโนมัติโดยใช้ AI สร้างสรรค์ ฉันเรียกสิ่งนี้ว่าโรงงานทางปัญญา

ฉันจำเป็นต้องพัฒนาโปรแกรมเพื่อนำกลไกนี้ไปใช้

ปัจจุบัน ฉันได้สร้างโปรแกรมที่สามารถทำให้การแปล การสร้างวิดีโอพรีเซนเทชัน และการอัปโหลดไปยัง YouTube เป็นไปโดยอัตโนมัติได้อย่างสมบูรณ์แล้ว

ตอนนี้ฉันกำลังอยู่ในกระบวนการสร้างโปรแกรมพื้นฐานสำหรับการจัดหมวดหมู่และเชื่อมโยงบทความบล็อกที่มีอยู่

เมื่อเสร็จสิ้น และฉันสร้างโปรแกรมเพื่อสร้างเว็บไซต์บล็อกของตัวเองและแสดงผลบนเว็บเซิร์ฟเวอร์โดยอัตโนมัติ แนวคิดเริ่มต้นของโรงงานทางปัญญาของฉันก็จะสมบูรณ์

โรงงานทางปัญญาในความหมายกว้าง

ฉบับร่างของบล็อกโพสต์ของฉัน ซึ่งทำหน้าที่เป็นวัตถุดิบสำหรับโรงงานทางปัญญานี้ ก็ได้รับการจัดการเป็นโครงการ GitHub เช่นกัน สำหรับตอนนี้ โครงการเหล่านี้เป็นแบบส่วนตัวและไม่เปิดเผยต่อสาธารณะ แต่ฉันกำลังพิจารณาที่จะทำให้เป็นโครงการสาธารณะพร้อมกับโปรแกรมโรงงานทางปัญญาในอนาคต

และการจัดหมวดหมู่บล็อกโพสต์ การเชื่อมโยงบทความ และการอธิบายบล็อกโพสต์ที่แปลงเป็นวิดีโอ ซึ่งฉันกำลังพัฒนาอยู่ในขณะนี้ ล้วนมีแนวคิดพื้นฐานเดียวกันกับ DeepWiki

การใช้ AI สร้างสรรค์ทำให้สามารถผลิตเนื้อหาต่างๆ ได้จากผลงานสร้างสรรค์ต้นฉบับเป็นวัตถุดิบ นอกจากนี้ ยังสามารถเชื่อมโยงข้อมูลและความรู้ภายในเนื้อหาเหล่านั้นได้อย่างมีประสิทธิภาพ สร้างเป็นฐานความรู้

ความแตกต่างเพียงอย่างเดียวคือวัตถุดิบเป็นโปรแกรมหรือบล็อกโพสต์ และสำหรับ DeepWiki และโรงงานทางปัญญาของฉันที่ขับเคลื่อนด้วย AI สร้างสรรค์ ความแตกต่างนั้นแทบจะไม่มีความหมายเลย

กล่าวอีกนัยหนึ่ง หากคำว่า "โรงงานทางปัญญา" ถูกตีความในความหมายทั่วไปที่กว้างขึ้น ไม่จำกัดเฉพาะโปรแกรมของฉัน DeepWiki ก็เป็นโรงงานทางปัญญาประเภทหนึ่งเช่นกัน

และสิ่งที่โรงงานทางปัญญาผลิตนั้นไม่จำกัดเฉพาะบทความที่แปลเป็นภาษาอื่น วิดีโอพรีเซนเทชัน เว็บไซต์บล็อกที่สร้างขึ้นเอง หรือเว็บไซต์ Wiki

พวกเขาน่าจะสามารถแปลงเนื้อหาให้เป็นสื่อและรูปแบบต่างๆ ที่สามารถจินตนาการได้ทั้งหมด เช่น วิดีโอสั้น ทวีต การ์ตูน แอนิเมชัน พอดแคสต์ และอีบุ๊ก

นอกจากนี้ เนื้อหาภายในสื่อและรูปแบบเหล่านี้ยังสามารถหลากหลายเพื่อตอบสนองผู้รับได้ เช่น การรองรับหลายภาษาที่กว้างขวางขึ้น เวอร์ชันสำหรับผู้เชี่ยวชาญหรือผู้เริ่มต้น และเวอร์ชันสำหรับผู้ใหญ่หรือเด็ก

ยิ่งไปกว่านั้น แม้กระทั่งการสร้างเนื้อหาที่ปรับแต่งตามความต้องการก็สามารถทำได้จริง

GitHub ในฐานะเหมืองทางปัญญา

วัตถุดิบสำหรับโรงงานทางปัญญานั้นโดยพื้นฐานแล้วสามารถอยู่ที่ใดก็ได้

อย่างไรก็ตาม เมื่อพิจารณาว่า GitHub ได้กลายเป็นมาตรฐานโดยพฤตินัยสำหรับการแบ่งปัน การแก้ไขร่วมกัน และการจัดเก็บโปรแกรมโครงการโอเพนซอร์ส และมีผู้คนจำนวนมาก ไม่ใช่แค่ฉันเท่านั้น ที่ใช้ GitHub เป็นที่เก็บเอกสาร ก็จะเห็นได้ชัดว่า GitHub มีศักยภาพที่จะเป็นแหล่งวัตถุดิบหลักสำหรับโรงงานทางปัญญา

กล่าวอีกนัยหนึ่ง GitHub จะกลายเป็นเหมืองทางปัญญาที่มนุษยชาติร่วมกันใช้ โดยจะจัดหาวัตถุดิบให้กับโรงงานทางปัญญา

คำว่า "ร่วมกันโดยมนุษยชาติ" ในที่นี้สะท้อนแนวคิดที่ว่าโครงการโอเพนซอร์สเป็นสินทรัพย์ซอฟต์แวร์ที่มนุษยชาติร่วมกันใช้

ปรัชญาโอเพนซอร์สที่สนับสนุน GitHub ก็จะเข้ากันได้ดีกับแนวคิดของเอกสารแบบเปิด

นอกจากนี้ วัฒนธรรมการจัดการข้อมูลลิขสิทธิ์และใบอนุญาตสำหรับเอกสารแต่ละฉบับที่คล้ายกับโปรแกรมก็อาจเกิดขึ้นได้ เนื้อหาที่สร้างขึ้นโดยอัตโนมัติจากเอกสารต้นฉบับสามารถกำหนดใบอนุญาตเดียวกันได้อย่างง่ายดาย หรือปฏิบัติตามกฎที่กำหนดโดยใบอนุญาต

จากมุมมองของการพัฒนาโรงงานทางปัญญา การรวมศูนย์เอกสารวัตถุดิบบน GitHub ถือเป็นสิ่งที่ดีที่สุด

สิ่งนี้ให้ประโยชน์สองประการ: ประสิทธิภาพการพัฒนาที่ดีขึ้นโดยเพียงแค่เชื่อมต่อ GitHub กับโรงงานทางปัญญา และความสามารถในการแสดงฟังก์ชันและประสิทธิภาพของโรงงานทางปัญญาของตนเองได้อย่างมีประสิทธิภาพโดยใช้เอกสารที่เปิดเผยต่อสาธารณะ คล้ายกับ DeepWiki

ในอนาคต เมื่อโรงงานทางปัญญาต่างๆ ได้รับการพัฒนาและสามารถเชื่อมต่อกับ GitHub ได้ และเมื่อผู้คนและบริษัทต่างๆ จัดการเอกสารบน GitHub และประมวลผลด้วยโรงงานทางปัญญามากขึ้น ตำแหน่งของ GitHub ในฐานะเหมืองทางปัญญาก็น่าจะได้รับการยอมรับอย่างมั่นคง

ฐานความรู้สาธารณะที่มนุษยชาติร่วมกันใช้

เมื่อ GitHub เป็นศูนย์กลางในฐานะเหมืองทางปัญญา และเนื้อหาและฐานความรู้ต่างๆ ถูกผลิตขึ้นโดยโรงงานทางปัญญา ระบบนิเวศทั้งหมดนี้จะสร้างฐานความรู้สาธารณะที่มนุษยชาติร่วมกันใช้

ยิ่งไปกว่านั้น มันเป็นฐานความรู้แบบไดนามิกและเรียลไทม์ที่จะขยายตัวโดยอัตโนมัติเมื่อจำนวนเอกสารที่เผยแพร่บน GitHub เพิ่มขึ้น

ในขณะที่ฐานความรู้ขนาดใหญ่และซับซ้อนนี้ ซึ่งประกอบด้วยความรู้มหาศาล จะเป็นประโยชน์ต่อมนุษย์ แต่ก็จะเป็นการยากที่จะดึงคุณค่าศักยภาพของมันออกมาได้อย่างเต็มที่

อย่างไรก็ตาม AI จะสามารถใช้ประโยชน์จากฐานความรู้สาธารณะนี้ ซึ่งมนุษยชาติทุกคนร่วมกันใช้ได้อย่างเต็มที่

สายแร่แห่งความรู้สาธารณะ

หากระบบนิเวศดังกล่าวเกิดขึ้นได้ ข้อมูลสาธารณะต่างๆ จะมารวมกันที่ GitHub โดยธรรมชาติ

สิ่งนี้จะไม่จำกัดอยู่แค่ฉบับร่างของบล็อกส่วนตัวหรือเว็บไซต์ขององค์กรเท่านั้น

ข้อมูลเชิงลึกและข้อมูลทางวิชาการ เช่น เอกสารที่ยังไม่ตีพิมพ์และแนวคิดการวิจัย ข้อมูลการทดลอง และผลสำรวจ ก็จะสะสมอยู่ด้วย

สิ่งนี้จะดึงดูดไม่เพียงแต่ผู้ที่ต้องการใช้ความรู้ แนวคิด และข้อมูลเพื่อประโยชน์ของมนุษยชาติทั้งหมดเท่านั้น แต่ยังรวมถึงผู้ที่ต้องการเผยแพร่การค้นพบของตนอย่างรวดเร็วและได้รับการยอมรับด้วย

แม้แต่นักวิชาการและนักวิจัย หลายคนก็จะพบคุณค่าในการที่ AI ตรวจสอบความถูกต้อง ความแปลกใหม่ และผลกระทบของผลงานของตน แสดงออกผ่านเนื้อหาที่หลากหลาย และได้รับการยอมรับในลักษณะที่แพร่กระจายไปในวงกว้าง โดยไม่ต้องรอขั้นตอนการตรวจสอบโดยผู้ทรงคุณวุฒิที่ใช้เวลานาน

หรืออีกทางหนึ่ง หากผลงานของพวกเขาดึงดูดความสนใจของนักวิจัยหรือบริษัทอื่นในลักษณะนี้ ซึ่งนำไปสู่การวิจัยร่วมกันหรือการระดมทุน ก็มีประโยชน์ในทางปฏิบัติด้วย

นอกจากนี้ ยังมีแนวโน้มที่จะมีการไหลกลับของความรู้ของ AI เองด้วย

AI สร้างสรรค์ได้รับความรู้จำนวนมหาศาลผ่านการฝึกอบรมเบื้องต้น แต่ไม่ได้สำรวจการเชื่อมโยงที่ไม่คาดคิดหรือโครงสร้างที่คล้ายกันระหว่างความรู้จำนวนมหาศาลนั้นอย่างกระตือรือร้นในระหว่างการเรียนรู้

เช่นเดียวกับข้อมูลเชิงลึกใหม่ๆ ที่เกิดขึ้นจากการเชื่อมโยงความรู้ที่แตกต่างกัน

ในทางกลับกัน เมื่ออธิบายความคล้ายคลึงและการเชื่อมโยงดังกล่าวในระหว่างการสนทนากับ AI สร้างสรรค์ที่ผ่านการฝึกอบรมเบื้องต้น ก็สามารถประเมินคุณค่าได้อย่างถูกต้องแม่นยำ

ดังนั้น การเปรียบเทียบและเชื่อมโยงความรู้ต่างๆ แบบสุ่มหรือครอบคลุมทั้งหมด และป้อนข้อมูลเหล่านั้นลงใน AI สร้างสรรค์ จึงเป็นไปได้ที่จะค้นพบความคล้ายคลึงที่ไม่คาดคิดและการเชื่อมโยงที่มีคุณค่า

แน่นอนว่าเนื่องจากมีชุดค่าผสมจำนวนมหาศาล จึงไม่สมจริงที่จะครอบคลุมทั้งหมด อย่างไรก็ตาม ด้วยการปรับปรุงและทำให้กระบวนการนี้เป็นอัตโนมัติอย่างเหมาะสม ก็จะสามารถค้นพบความรู้ที่เป็นประโยชน์จากความรู้ที่มีอยู่ได้โดยอัตโนมัติ

ด้วยการบรรลุการค้นพบความรู้โดยอัตโนมัติและการจัดเก็บความรู้ที่ค้นพบนั้นบน GitHub ดูเหมือนว่าจะสามารถทำซ้ำวงจรนี้ได้อย่างไม่มีกำหนด

ด้วยวิธีนี้ แหล่งความรู้ที่ยังไม่ถูกค้นพบจำนวนมากจึงมีอยู่ในเหมืองทางปัญญานี้ และจะสามารถขุดค้นพบได้

บทสรุป

เมื่อฐานความรู้ของมนุษย์ที่ใช้ร่วมกันเป็นมาตรฐานโดยพฤตินัยเช่น GitHub ได้รับการจัดตั้งขึ้น ก็มีแนวโน้มที่จะถูกนำไปใช้สำหรับการฝึกอบรม AI สร้างสรรค์ล่วงหน้า และสำหรับการดึงข้อมูลความรู้ เช่น RAG

ในสถานการณ์นั้น GitHub เองจะทำหน้าที่เหมือนสมองขนาดใหญ่ และ AI สร้างสรรค์จะแบ่งปันสมองนี้ กระจายและขยายความรู้ในขณะที่แบ่งปัน

ความรู้ที่บันทึกเพิ่มเติมในที่นั้นจะไม่เพียงแต่รวมถึงบันทึกข้อเท็จจริง ข้อมูลใหม่ หรือการจัดหมวดหมู่เท่านั้น แต่อาจรวมถึงความรู้เชิงเร่งปฏิกิริยาที่ส่งเสริมการค้นพบความรู้อื่นๆ หรือการผสมผสานใหม่ๆ

ฉันเรียกความรู้ที่มีผลเชิงเร่งปฏิกิริยาเช่นนี้ว่า "ผลึกทางปัญญา" หรือ "ผลึกความรู้" ซึ่งรวมถึง ตัวอย่างเช่น กรอบความคิดใหม่ๆ

เมื่อกรอบความคิดถูกค้นพบหรือพัฒนาขึ้นใหม่ และมีการเพิ่มผลึกทางปัญญาเข้าไป ผลเชิงเร่งปฏิกิริยาของมันจะช่วยให้สามารถผสมผสานและจัดโครงสร้างความรู้ในรูปแบบที่แตกต่างจากเดิม นำไปสู่การเติบโตของความรู้ใหม่

ในบรรดาเหล่านี้ อาจมีผลึกความรู้อื่นๆ ซึ่งจะช่วยเพิ่มพูนความรู้ให้มากยิ่งขึ้นไปอีก

ความรู้ดังกล่าวไม่ใช่การค้นพบทางวิทยาศาสตร์ แต่เป็นสิ่งที่ใกล้เคียงกับการสอบสวนทางคณิตศาสตร์ การพัฒนาทางวิศวกรรม หรือการประดิษฐ์ ดังนั้น จึงเป็นความรู้ที่เติบโตขึ้นจากการคิดล้วนๆ มากกว่าการได้มาซึ่งข้อเท็จจริงจากการสังเกตการณ์ใหม่ๆ เช่นความรู้ทางวิทยาศาสตร์

และ GitHub ในฐานะเหมืองทางปัญญา พร้อมด้วย AI สร้างสรรค์จำนวนนับไม่ถ้วนที่ใช้ประโยชน์จากมัน จะเร่งการเติบโตของความรู้ดังกล่าว

ความรู้ที่ถูกค้นพบอย่างต่อเนื่องในอัตราที่เกินกว่าขีดความสามารถในการค้นพบของมนุษย์ จะถูกนำเสนอในรูปแบบที่เข้าใจง่ายโดยโรงงานความรู้

ด้วยวิธีนี้ ความรู้ที่สามารถสำรวจได้ด้วยความคิดล้วนๆ จะถูกขุดค้นพบอย่างรวดเร็ว
