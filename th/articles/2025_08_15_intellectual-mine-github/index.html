GitHub ในฐานะเหมืองทางปัญญา

15 สิงหาคม 2568

คุณรู้จัก GitHub ซึ่งเป็นบริการเว็บที่นักพัฒนาซอฟต์แวร์โอเพนซอร์สใช้เป็นแพลตฟอร์มสำหรับการพัฒนาร่วมกันหรือไม่?

ในช่วงไม่กี่ปีที่ผ่านมา การใช้งานในฐานะพื้นที่ทำงานร่วมกันได้ขยายออกไปนอกเหนือจากซอฟต์แวร์โอเพนซอร์ส ไปสู่การพัฒนาซอฟต์แวร์ขององค์กร และแม้แต่แอปพลิเคชันที่ไม่เกี่ยวข้องกับซอฟต์แวร์

ฉันก็ใช้ GitHub เพื่อจัดการโปรแกรมของตัวเองและร่างบทความที่ฉันเขียนสำหรับบล็อกนี้เช่นกัน

ในบทความนี้ ฉันจะสำรวจความเป็นไปได้ที่การใช้งาน GitHub จะขยายออกไปนอกเหนือจากซอฟต์แวร์มากขึ้นเรื่อยๆ กลายเป็นพื้นที่สำหรับการแบ่งปันความรู้แบบเปิด

การสร้างเว็บไซต์ Wiki โดย DeepWiki

เครื่องมือพัฒนาซอฟต์แวร์จำนวนมากที่ใช้ AI เชิงสร้างสรรค์ ถูกออกแบบมาเพื่อช่วยเหลือนักเขียนโปรแกรมที่เป็นมนุษย์ ในเครื่องมือเหล่านี้ มนุษย์จะเป็นผู้เขียนโปรแกรม และ AI จะให้การสนับสนุน

อย่างไรก็ตาม เครื่องมือพัฒนาซอฟต์แวร์ประเภทใหม่กำลังเกิดขึ้น โดยที่มนุษย์เพียงแค่ให้คำแนะนำ และ AI เชิงสร้างสรรค์จะรับหน้าที่ในการสร้างโปรแกรม

หนึ่งในเครื่องมือบุกเบิกที่ได้รับความสนใจคือ Devin บางคนกล่าวว่าการนำ Devin มาใช้ก็เหมือนกับการเพิ่มโปรแกรมเมอร์อีกคนเข้าสู่ทีมพัฒนา แม้จะยังกล่าวกันว่าวิศวกรที่เป็นมนุษย์จำเป็นต้องให้การสนับสนุนอย่างละเอียดเพื่อการใช้งานที่มีประสิทธิภาพ แต่ข้อมูลดังกล่าวจะถูกรวบรวมและนำไปใช้ในการปรับปรุงต่อไปอย่างไม่ต้องสงสัย

ยุคที่ทีมพัฒนาซอฟต์แวร์โดยทั่วไปประกอบด้วยมนุษย์หนึ่งคนและโปรแกรมเมอร์ AI อย่าง Devin เป็นสมาชิกทีมกำลังใกล้เข้ามาอย่างรวดเร็ว

Cognition ผู้พัฒนา Devin ยังได้เปิดตัวบริการที่เรียกว่า DeepWiki

DeepWiki เป็นบริการที่สร้างเว็บไซต์ wiki สำหรับโครงการพัฒนาซอฟต์แวร์แต่ละโครงการบน GitHub โดยอัตโนมัติ ซึ่งหมายความว่า AI อย่าง Devin จะอ่านและวิเคราะห์โปรแกรมทั้งหมดและเอกสารที่เกี่ยวข้องของโครงการ จากนั้นจะสร้างเอกสารและข้อกำหนดการออกแบบทั้งหมด

Cognition รายงานว่าได้สร้างเว็บไซต์ wiki สำหรับโครงการพัฒนาซอฟต์แวร์สาธารณะหลักๆ กว่า 50,000 โครงการบน GitHub ซึ่งทุกคนสามารถเข้าถึงได้อย่างอิสระ

เนื่องจากเป็นโครงการสาธารณะ จึงไม่มีปัญหาในการดำเนินการดังกล่าว แม้ว่าเว็บไซต์ wiki สามารถสร้างขึ้นได้โดยอัตโนมัติ แต่จะต้องใช้ AI เชิงสร้างสรรค์จำนวนมากทำงานเต็มกำลังเป็นระยะเวลานาน ทำให้เกิดค่าใช้จ่ายจำนวนมาก

การที่ Cognition รับผิดชอบค่าใช้จ่ายเหล่านี้ ทำให้โครงการสาธารณะจำนวนมากได้รับประโยชน์จากการได้รับเอกสารและข้อกำหนดการออกแบบโดยไม่เสียค่าใช้จ่าย

หากข้อมูลทางสถิติแสดงให้เห็นว่าเว็บไซต์ wiki เหล่านี้มีประโยชน์สำหรับโครงการสาธารณะ และมีผลกระทบอย่างมีนัยสำคัญต่อการปรับปรุงคุณภาพและประสิทธิภาพ บริษัทพัฒนาซอฟต์แวร์ก็มีแนวโน้มที่จะนำ DeepWiki มาใช้สำหรับโครงการของตนเอง

Cognition ต้องลงทุนในการสร้างเว็บไซต์ wiki สำหรับโครงการสาธารณะจำนวนมาก โดยเชื่อว่าสิ่งนี้จะเกิดขึ้น สิ่งนี้แสดงให้เห็นถึงความมั่นใจของ Cognition ใน DeepWiki และหาก DeepWiki ถูกนำมาใช้ Devin ก็จะตามมาโดยอัตโนมัติ ซึ่งจะเร่งการแพร่หลายของโปรแกรมเมอร์ AI อย่างมีนัยสำคัญ

GitHub ในฐานะแพลตฟอร์มการแบ่งปันเอกสาร

GitHub ได้กลายเป็นบริการเว็บยอดนิยมและเป็นมาตรฐานโดยพฤตินัยสำหรับการแบ่งปัน แก้ไขร่วมกัน และจัดเก็บโปรแกรมสำหรับการพัฒนาซอฟต์แวร์โอเพนซอร์ส

ในช่วงไม่กี่ปีที่ผ่านมา คุณสมบัติการจัดการและความปลอดภัยที่แข็งแกร่งสำหรับองค์กร ทำให้บริษัทพัฒนาซอฟต์แวร์ที่ก้าวหน้าหันมาใช้งานเป็นเรื่องปกติ

ด้วยเหตุนี้ GitHub จึงมักถูกมองว่าเป็นบริการเว็บสำหรับจัดเก็บและแบ่งปันโปรแกรมเป็นหลัก อย่างไรก็ตาม ในความเป็นจริงแล้ว GitHub สามารถใช้แบ่งปัน แก้ไขร่วมกัน และจัดเก็บเอกสารและเนื้อหาต่างๆ ที่ไม่เกี่ยวข้องกับโปรแกรมได้เลย

ด้วยเหตุนี้ หลายคนจึงใช้ GitHub เพื่อจัดการเอกสารที่ต้องการแก้ไขร่วมกันอย่างกว้างขวาง เอกสารเหล่านี้อาจเกี่ยวข้องกับซอฟต์แวร์หรือไม่เกี่ยวข้องเลยก็ได้

นอกจากนี้ บล็อกและเว็บไซต์ก็เป็นเอกสารที่ประกอบด้วยโปรแกรมบางประเภท หรือถูกจัดโครงสร้างโดยโปรแกรมเพื่อเผยแพร่

ดังนั้น จึงไม่ใช่เรื่องแปลกสำหรับบุคคลและบริษัทที่จะจัดเก็บเนื้อหาบล็อกและเว็บไซต์ พร้อมด้วยโปรแกรมสำหรับการนำเสนอและโปรแกรมสำหรับการสร้างเว็บไซต์อัตโนมัติ ไว้รวมกันเป็นโครงการ GitHub เดียว

ยังสามารถทำให้เนื้อหาบล็อกและเว็บไซต์ดังกล่าวเป็นโครงการ GitHub สาธารณะเพื่อเปิดใช้งานการแก้ไขร่วมกันได้อีกด้วย

เมื่อเร็วๆ นี้ นอกเหนือจากการใช้ AI เชิงสร้างสรรค์สำหรับการพัฒนาซอฟต์แวร์แล้ว ยังมีการฝังฟังก์ชันการทำงานของ AI เชิงสร้างสรรค์ลงในซอฟต์แวร์โดยตรงเพิ่มมากขึ้น

ในกรณีเช่นนี้ คำสั่งโดยละเอียดสำหรับ AI เชิงสร้างสรรค์ ซึ่งเรียกว่าพรอมต์ จะถูกฝังอยู่ภายในโปรแกรม

พรอมต์เหล่านี้ก็สามารถถือเป็นเอกสารประเภทหนึ่งได้เช่นกัน

โรงงานทางปัญญา

แม้ว่าฉันจะเป็นวิศวกรซอฟต์แวร์ แต่ฉันก็เขียนบทความสำหรับบล็อกของฉันด้วย

ในขณะที่ฉันต้องการให้มีคนอ่านจำนวนมาก การเพิ่มจำนวนผู้อ่านนั้นค่อนข้างท้าทาย

แน่นอนว่า ฉันอาจพิจารณาสร้างบทความเพื่อดึงดูดความสนใจ หรือติดต่อโดยตรงกับบุคคลผู้มีอิทธิพลต่างๆ เพื่อขอคำแนะนำ โดยใช้ความพยายามและไหวพริบ

อย่างไรก็ตาม เมื่อพิจารณาจากบุคลิกของฉัน รวมถึงความพยายามและความเครียดที่เกี่ยวข้อง ฉันจึงไม่กระตือรือร้นกับการโปรโมทที่ก้าวร้าว ยิ่งไปกว่านั้น การใช้เวลาไปกับกิจกรรมดังกล่าวจะทำให้เสียเวลาจากส่วนหลักของงานของฉัน นั่นคือ การสร้างโปรแกรม การคิด และการเขียนเอกสาร

ดังนั้น เมื่อเร็วๆ นี้ ฉันจึงตัดสินใจลองใช้กลยุทธ์ "มัลติมีเดีย" หรือ "ออมนิแชนแนล" เพื่อขยายขอบเขตการเข้าถึงบทความบล็อกของฉัน โดยการนำเสนอในรูปแบบเนื้อหาที่หลากหลาย

โดยเฉพาะอย่างยิ่ง นี่คือการแปลบทความภาษาญี่ปุ่นเป็นภาษาอังกฤษและโพสต์ลงในเว็บไซต์บล็อกภาษาอังกฤษ และสร้างวิดีโอการนำเสนอเพื่ออธิบายบทความและเผยแพร่บน YouTube

นอกจากนี้ นอกเหนือจากการเผยแพร่บนบริการบล็อกทั่วไปแล้ว ฉันยังพิจารณาที่จะสร้างเว็บไซต์บล็อกของตัวเองพร้อมดัชนีบทความเก่าๆ ของฉันตามหมวดหมู่ และเชื่อมโยงบทความที่เกี่ยวข้องอีกด้วย

หากฉันต้องสร้างสิ่งเหล่านี้ทั้งหมดด้วยตนเองทุกครั้งที่เพิ่มบทความใหม่ มันก็จะผิดวัตถุประสงค์ ดังนั้น งานทั้งหมด ยกเว้นการเขียนบทความภาษาญี่ปุ่นฉบับแรก จะถูกทำให้เป็นอัตโนมัติโดยใช้ AI เชิงสร้างสรรค์ ฉันเรียกสิ่งนี้ว่า โรงงานทางปัญญา

ฉันจำเป็นต้องพัฒนาโปรแกรมเพื่อทำให้ระบบนี้เป็นจริง

ปัจจุบัน ฉันได้สร้างโปรแกรมที่สามารถทำให้การแปล การสร้างวิดีโอการนำเสนอ และการอัปโหลด YouTube เป็นไปโดยอัตโนมัติได้อย่างสมบูรณ์แล้ว

ตอนนี้ ฉันกำลังอยู่ในกระบวนการสร้างโปรแกรมพื้นฐานสำหรับการจัดหมวดหมู่และเชื่อมโยงบทความบล็อกที่มีอยู่

เมื่อเสร็จสิ้น และฉันสร้างโปรแกรมเพื่อสร้างเว็บไซต์บล็อกที่กำหนดเองและปรับใช้กับเว็บเซิร์ฟเวอร์โดยอัตโนมัติ แนวคิดเริ่มต้นของ โรงงานทางปัญญา ของฉันก็จะบรรลุผลโดยสมบูรณ์

โรงงานทางปัญญาในความหมายที่กว้างขึ้น

ร่างบทความบล็อกของฉัน ซึ่งทำหน้าที่เป็นวัตถุดิบสำหรับโรงงานทางปัญญานี้ ยังถูกจัดการในฐานะโครงการ GitHub ปัจจุบันยังไม่ได้เปิดเผยต่อสาธารณะในฐานะโครงการส่วนตัว แต่ฉันกำลังพิจารณาที่จะทำให้เป็นโครงการสาธารณะในอนาคต พร้อมกับโปรแกรมของโรงงานทางปัญญา

นอกจากนี้ การจัดหมวดหมู่บทความบล็อก การเชื่อมโยงบทความ และคำอธิบายวิดีโอของบทความบล็อกที่ฉันกำลังพัฒนาอยู่ในขณะนี้ ล้วนมีแนวคิดพื้นฐานเดียวกันกับ DeepWiki

ด้วยการใช้ AI เชิงสร้างสรรค์ ผลงานสร้างสรรค์ต้นฉบับจะถูกใช้เป็นวัตถุดิบในการผลิตเนื้อหาที่หลากหลาย นอกจากนี้ ข้อมูลและความรู้ภายในเนื้อหาเหล่านี้ยังสามารถเชื่อมโยงกันเพื่อสร้างสิ่งที่เรียกว่าฐานความรู้ได้

ความแตกต่างเพียงอย่างเดียวอยู่ที่ว่าวัตถุดิบนั้นเป็นโปรแกรมหรือบทความบล็อก และสำหรับ DeepWiki และโรงงานทางปัญญาของฉัน ซึ่งขับเคลื่อนโดย AI เชิงสร้างสรรค์ ความแตกต่างนี้ไม่สำคัญนัก

กล่าวอีกนัยหนึ่ง หากตีความคำว่า "โรงงานทางปัญญา" ในความหมายทั่วไปที่กว้างขึ้น โดยไม่จำกัดเฉพาะโปรแกรมของฉัน DeepWiki ก็เป็นโรงงานทางปัญญาประเภทหนึ่งเช่นกัน

ยิ่งไปกว่านั้น สิ่งที่โรงงานทางปัญญาผลิตไม่ได้จำกัดอยู่แค่บทความที่แปลเป็นภาษาอื่น วิดีโอการนำเสนอ หรือเว็บไซต์บล็อกและ wiki ที่สร้างขึ้นเอง

มีแนวโน้มที่จะสามารถแปลงเนื้อหาไปเป็นสื่อและรูปแบบทุกประเภทเท่าที่จะเป็นไปได้ เช่น วิดีโอสั้น ทวีต มังงะและอนิเมะ พอดแคสต์ และอีบุ๊ก

นอกจากนี้ เนื้อหาภายในสื่อและรูปแบบเหล่านี้ยังสามารถทำให้มีความหลากหลายเพื่อให้เหมาะกับผู้ชมที่แตกต่างกัน ซึ่งรวมถึงการทำให้เป็นหลายภาษาในวงกว้างขึ้น เวอร์ชันสำหรับผู้เชี่ยวชาญหรือผู้เริ่มต้น และเวอร์ชันสำหรับผู้ใหญ่หรือเด็ก

ท้ายที่สุดแล้ว แม้แต่การสร้างเนื้อหาที่ปรับแต่งตามความต้องการก็ยังเป็นไปได้

GitHub ในฐานะเหมืองทางปัญญา

วัตถุดิบสำหรับโรงงานทางปัญญา โดยหลักการแล้วสามารถจัดเก็บไว้ที่ใดก็ได้

อย่างไรก็ตาม เมื่อพิจารณาว่า GitHub ได้กลายเป็นมาตรฐานโดยพฤตินัยสำหรับการแบ่งปัน แก้ไขร่วมกัน และจัดเก็บโปรแกรมสำหรับโครงการโอเพนซอร์ส และบุคคลต่างๆ — ไม่ใช่แค่ตัวฉันเอง — ใช้ GitHub เป็นที่จัดเก็บเอกสาร จึงเห็นได้ชัดว่า GitHub มีศักยภาพที่จะเป็นแหล่งวัตถุดิบหลักสำหรับโรงงานทางปัญญา

กล่าวอีกนัยหนึ่ง GitHub จะกลายเป็นเหมืองทางปัญญาที่มนุษยชาติแบ่งปัน โดยจัดหาวัตถุดิบให้กับโรงงานทางปัญญา

คำว่า "มนุษยชาติแบ่งปัน" ในที่นี้ สะท้อนถึงแนวคิดที่ว่าโครงการโอเพนซอร์สเป็นสินทรัพย์ซอฟต์แวร์ที่มนุษยชาติแบ่งปันกัน

ปรัชญาโอเพนซอร์สที่เป็นรากฐานของ GitHub จะเข้ากันได้ดีกับแนวคิดของเอกสารเปิด

นอกจากนี้ วัฒนธรรมการจัดการข้อมูลลิขสิทธิ์และใบอนุญาตสำหรับเอกสารแต่ละฉบับ คล้ายกับโปรแกรม อาจเกิดขึ้นได้ เนื้อหาที่สร้างขึ้นโดยอัตโนมัติจากเอกสารต้นฉบับสามารถกำหนดใบอนุญาตเดียวกัน หรือปฏิบัติตามกฎที่กำหนดโดยใบอนุญาตได้อย่างง่ายดาย

จากมุมมองของการสร้างโรงงานทางปัญญา การรวมเอกสารวัตถุดิบไว้ใน GitHub นั้นเป็นสิ่งที่ดีที่สุด

สิ่งนี้ให้ประโยชน์สองประการ: ประโยชน์ด้านประสิทธิภาพการพัฒนา เนื่องจากการเชื่อมต่อ GitHub เข้ากับโรงงานทางปัญญาเป็นเรื่องง่าย และความสามารถในการแสดงให้เห็นถึงฟังก์ชันและประสิทธิภาพของโรงงานทางปัญญาของตนเองต่อเอกสารที่เผยแพร่ต่อสาธารณะได้อย่างมีประสิทธิภาพ คล้ายกับ DeepWiki

ในอนาคต เมื่อโรงงานทางปัญญาต่างๆ ได้รับการพัฒนาและเชื่อมต่อกับ GitHub และบุคคลและบริษัทจำนวนมากขึ้นจัดการเอกสารบน GitHub เพื่อประมวลผลโดยโรงงานทางปัญญา ตำแหน่งของ GitHub ในฐานะเหมืองทางปัญญาควรจะมั่นคงขึ้น

ฐานความรู้สาธารณะที่มนุษยชาติร่วมกัน

ด้วย GitHub เป็นแกนหลัก ซึ่งทำหน้าที่เป็นเหมืองทางปัญญา และโรงงานทางปัญญาที่ผลิตเนื้อหาและฐานความรู้ที่หลากหลาย ระบบนิเวศทั้งหมดนี้จะสร้างฐานความรู้สาธารณะที่มนุษยชาติร่วมกัน

ยิ่งไปกว่านั้น นี่จะเป็นฐานความรู้แบบไดนามิกที่ทำงานแบบเรียลไทม์ ซึ่งจะขยายตัวโดยอัตโนมัติเมื่อจำนวนเอกสารที่เผยแพร่บน GitHub เพิ่มขึ้น

แม้ว่าฐานความรู้ที่ซับซ้อนและใหญ่โตมหาศาลนี้ ซึ่งบรรจุความรู้จำนวนมาก จะเป็นประโยชน์ต่อมนุษย์ แต่การดึงคุณค่าศักยภาพของมันออกมาอย่างเต็มที่น่าจะเป็นเรื่องท้าทายสำหรับเรา

อย่างไรก็ตาม AI จะสามารถใช้ประโยชน์จากฐานความรู้สาธารณะที่มนุษยชาติร่วมกันนี้ได้อย่างเต็มที่

สายธารความรู้สาธารณะ

เมื่อระบบนิเวศดังกล่าวเป็นจริง ข้อมูลสาธารณะต่างๆ จะมารวมกันที่ GitHub โดยธรรมชาติ

สิ่งนี้จะไม่จำกัดอยู่แค่ฉบับร่างของบล็อกส่วนตัวหรือเว็บไซต์ของบริษัทเท่านั้น

ข้อมูลเชิงลึกทางวิชาการและข้อมูล เช่น เอกสารก่อนตีพิมพ์ แนวคิดการวิจัย ข้อมูลการทดลอง และผลการสำรวจ ก็จะถูกรวบรวมไว้ที่นั่นเช่นกัน

สิ่งนี้จะไม่เพียงดึงดูดผู้ที่ต้องการบริจาคความรู้ แนวคิด และข้อมูลเพื่อประโยชน์ของมนุษยชาติเท่านั้น แต่ยังดึงดูดผู้ที่ต้องการเผยแพร่การค้นพบอย่างรวดเร็วเพื่อได้รับการยอมรับอีกด้วย

แม้แต่นักวิชาการและนักวิจัยก็อาจพบคุณค่าในการให้ AI ตรวจสอบความถูกต้อง ความแปลกใหม่ และผลกระทบของผลงาน โดยแสดงออกผ่านรูปแบบเนื้อหาต่างๆ และได้รับการยอมรับโดย "การแพร่กระจายอย่างรวดเร็ว" แทนที่จะต้องรอขั้นตอนการตรวจสอบโดยผู้ทรงคุณวุฒิที่ยาวนานและใช้เวลานาน

หรืออีกทางหนึ่ง หากผลงานของพวกเขาดึงดูดความสนใจของนักวิจัยหรือบริษัทอื่นๆ ในลักษณะนี้ ซึ่งนำไปสู่การวิจัยร่วมกันหรือการระดมทุน ก็จะมีประโยชน์ที่เป็นรูปธรรม

นอกจากนี้ ยังจะมีการหมุนเวียนความรู้ของ AI เองด้วย

ในขณะที่ AI เชิงสร้างสรรค์ได้รับความรู้จำนวนมากจากการฝึกอบรมล่วงหน้า แต่ก็ไม่ได้เรียนรู้อย่างกระตือรือร้นโดยการสำรวจความเชื่อมโยงที่ไม่คาดคิดหรือโครงสร้างที่คล้ายกันในหมู่ความรู้จำนวนมหาศาลนั้น

เช่นเดียวกันกับข้อมูลเชิงลึกใหม่ๆ ที่เกิดขึ้นจากการเชื่อมโยงความรู้ที่แตกต่างกัน

ในทางกลับกัน เมื่ออภิปรายความคล้ายคลึงและความเชื่อมโยงดังกล่าวกับ AI เชิงสร้างสรรค์ที่ได้รับการฝึกอบรมล่วงหน้า AI สามารถประเมินคุณค่าของสิ่งเหล่านั้นได้อย่างแม่นยำทีเดียว

ดังนั้น ด้วยการป้อนความรู้ต่างๆ เข้าไปใน AI เชิงสร้างสรรค์ และเปรียบเทียบแบบสุ่มหรืออย่างละเอียดถี่ถ้วน ก็เป็นไปได้ที่จะค้นพบความคล้ายคลึงที่ไม่คาดคิดและความเชื่อมโยงที่มีคุณค่า

แน่นอนว่า ด้วยจำนวนการผสมผสานที่มหาศาล การครอบคลุมทุกอย่างเป็นเรื่องที่ไม่สามารถทำได้จริง อย่างไรก็ตาม ด้วยการปรับปรุงและทำให้กระบวนการนี้เป็นอัตโนมัติอย่างเหมาะสม ก็เป็นไปได้ที่จะขุดค้นความรู้ที่เป็นประโยชน์จากความรู้ที่มีอยู่โดยอัตโนมัติ

ด้วยการบรรลุการค้นพบความรู้โดยอัตโนมัติดังกล่าว และการจัดเก็บความรู้ที่ค้นพบใน GitHub วงจรนี้ดูเหมือนจะสามารถดำเนินต่อไปได้อย่างไม่มีกำหนด

ดังนั้น ภายในเหมืองทางปัญญานี้ มีสายธารที่ยังไม่ถูกค้นพบจำนวนมาก และจะสามารถขุดค้นพบสิ่งเหล่านั้นได้

บทสรุป

ในขณะที่ฐานความรู้สาธารณะที่มนุษยชาติร่วมกัน ซึ่งเป็นมาตรฐานโดยพฤตินัยอย่าง GitHub ก่อตัวขึ้นในลักษณะนี้ มีแนวโน้มที่จะถูกนำไปใช้สำหรับการฝึกอบรมล่วงหน้า AI เชิงสร้างสรรค์ และสำหรับกลไกการดึงความรู้ เช่น RAG

ในสถานการณ์เช่นนี้ GitHub เองจะทำหน้าที่เหมือนสมองขนาดใหญ่ และ AI เชิงสร้างสรรค์จะแบ่งปันสมองนี้ โดยกระจายและขยายความรู้

ความรู้ที่บันทึกเพิ่มเติมที่นั่นจะไม่ใช่เพียงแค่บันทึกข้อเท็จจริง ข้อมูลใหม่ หรือการจัดหมวดหมู่เท่านั้น แต่ยังรวมถึงความรู้ที่ทำหน้าที่เป็นตัวเร่ง ซึ่งส่งเสริมการค้นพบความรู้อื่นๆ และการผสมผสานใหม่ๆ

ฉันเรียกความรู้ที่มีผลเร่งปฏิกิริยาดังกล่าวว่า ผลึกทางปัญญา หรือผลึกความรู้ ซึ่งรวมถึงตัวอย่างเช่น กรอบแนวคิดใหม่ๆ

เมื่อกรอบแนวคิดถูกค้นพบหรือพัฒนาขึ้นใหม่ และมีการเพิ่มผลึกทางปัญญา ผลเร่งปฏิกิริยาของมันจะช่วยให้เกิดการผสมผสานและการจัดโครงสร้างความรู้แบบใหม่ที่ไม่เคยมีมาก่อน นำไปสู่การเพิ่มขึ้นของความรู้ใหม่ๆ

บางครั้งสิ่งเหล่านี้อาจมีผลึกทางปัญญาอีกชิ้นหนึ่ง ซึ่งจากนั้นจะขยายความรู้ให้กว้างขวางยิ่งขึ้น

ความรู้ประเภทนี้ใกล้เคียงกับการสืบค้นทางคณิตศาสตร์ การพัฒนาทางวิศวกรรม หรือการประดิษฐ์ มากกว่าการค้นพบทางวิทยาศาสตร์ ดังนั้นจึงเป็นความรู้ที่เติบโตขึ้นจากการคิดล้วนๆ มากกว่าจากข้อเท็จจริงจากการสังเกตการณ์ใหม่ๆ เช่นความรู้ทางวิทยาศาสตร์

และ GitHub ในฐานะเหมืองทางปัญญา พร้อมด้วย AI เชิงสร้างสรรค์จำนวนนับไม่ถ้วนที่ใช้ประโยชน์จากมัน จะเร่งการเติบโตของความรู้ดังกล่าว

ความรู้ที่ถูกค้นพบอย่างรวดเร็วนี้ ซึ่งเกินกว่าอัตราการค้นพบของมนุษย์อย่างมาก จะถูกจัดให้อยู่ในรูปแบบที่เข้าใจง่ายโดยโรงงานทางปัญญา

ด้วยวิธีนี้ ความรู้ที่สามารถสำรวจได้ด้วยการคิดล้วนๆ จะถูกขุดค้นพบอย่างรวดเร็ว
